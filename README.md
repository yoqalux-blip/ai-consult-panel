# AI Consult Panel Skill

`ai-consult-panel` 是一个通用 Codex Skill，用于通过用户已经登录的 Chrome 网页会话，把 ChatGPT、Gemini、Qwen、DeepSeek、Kimi、秘塔作为外部顾问面板调用。

它不是 API 集成，也不是网页自动化脚本库。v1 的重点是稳定的工作流、提示模板、模型分工、信任权重和综合判断规范。外部模型负责扩宽思路、提供反方质疑、补充中文网页/学术线索、生成视觉/写作/代码方案；最终判断、验证和落地仍由 Codex 负责。

## 核心原则

- Codex 是主决策者，外部模型只是内容扩充和二次思考工具。
- 不把六个模型当投票器；不按多数意见自动决定。
- 重要事实、文献、指南、DOI、API、法律/医疗/金融建议都必须复核。
- 不发送密码、cookie、本地存储、账号安全信息、密钥或可识别个人隐私数据。
- 涉及真实敏感数据时，先去标识化或只发送聚合描述。
- Pro、Deep Research、秘塔深度研究、图片生成、长任务和点数消耗模式默认视为高消耗；除非用户已明确授权，否则先询问。

## 支持模型与定位

| 模型 | 默认定位 | 主要用途 | 注意事项 |
| --- | --- | --- | --- |
| ChatGPT | 主要推理锚点 | 保守审稿、长上下文项目、写作重构、方法学/代码/架构批判、研究/图片模式 | Pro 或高额度模式需确认 |
| Gemini | 第二外部视角 | 发散构思、视觉/图表/叙事、跨领域启发、Deep Research、图片生成 | 容易过度赞同，需折扣处理 |
| Qwen | 中文与项目补充 | 中文/双语表达、编码/项目替代方案、中文网页感知 | 不作为最终事实源 |
| DeepSeek | 逻辑红队 | 数学、代码、调试、简洁推理、失败模式审查 | 不作为唯一证据源 |
| Kimi | 中文长文档与制品 | 长文档、PPT、表格、网站、K2.6 思考、结构化制品规划 | 当前按非会员能力处理 |
| Metaso/秘塔 | 来源发现引擎 | 中文网页、学术、报告、脑图、来源清单、深度研究 | 深度研究消耗点数和时间，需谨慎 |

## 信任权重

优先级不是模型名气投票，而是证据质量：

1. 本地已验证证据、原始资料、PDF 原文、官方文档。
2. Codex 可复现的检查、代码运行、数据核验和推理。
3. ChatGPT 的保守批判和长上下文综合。
4. Gemini 的创造性方案，但要扣除阿谀、过度乐观和未证实断言。
5. Qwen、DeepSeek、Kimi、秘塔作为补充视角、中文来源线索、表达方案和红队意见。

## 什么时候调用

适合调用：

- 研究设计、统计分析计划、方法学风险、数据泄漏、校准、验证、DCA。
- 文献综述、证据矩阵、论文写作、答辩、基金、报告、产品/设计叙事。
- 图表、视觉解释、生成图片方向、PPT 页面结构。
- 代码架构、调试策略、测试设计、重构权衡。
- Codex 自觉思路变窄、缺少替代方案或需要反方审稿时。

不建议调用：

- 简单文件整理、机械格式转换、明确事实查询、运行测试、套用已确定编辑。
- 已有本地证据足够且下一步是确定性操作。

## 并发/子代理规则

当用户明确要求并发或六模型面板时，Codex 父代理负责拆解任务和最终综合。每个 worker 只负责一个模型或一个清晰子问题，不抢占其他标签页，不消耗高额度模式，不替 Codex 做最终判断。

若 Chrome 多标签并行不稳定，就串行输入浏览器任务，同时并行做提示词准备、结果抽取、证据核验和综合判断。

## 本地安装

将本仓库内容放到：

```text
C:\Users\Administrator\.codex\skills\ai-consult-panel
```

或复制到任意 Codex skills 目录下的 `ai-consult-panel` 文件夹。

## 调用示例

```text
用 $ai-consult-panel 让 ChatGPT 和 Gemini 对这个论文图表方案做外部审稿。
```

```text
用 $ai-consult-panel 开六模型面板，但不要触发 ChatGPT Pro，不要用秘塔深度研究。
```

```text
用 $ai-consult-panel 调 Kimi 做中文 PPT 结构，DeepSeek 做逻辑红队，最后你来综合。
```
